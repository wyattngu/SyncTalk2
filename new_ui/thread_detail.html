Hey team, following up on our sync last week regarding the frontend performance bottlenecks in the main dashboard. As discussed, the current Redux implementation is adding significant overhead, especially with the deep nesting in the user preferences state.

I've been evaluating Zustand as a lightweight alternative, and the initial POC looks very promising. The boilerplate reduction alone is massive.

Current Redux Example

// Reducer boilerplate
const initialState = { loading: false, data: null };
function userReducer(state = initialState, action) {
  switch (action.type) {
    case 'FETCH_START': return { ...state, loading: true };
    case 'FETCH_SUCCESS': return { loading: false, data: action.payload };
    default: return state;
  }
}

Zustand Equivalent

// Much cleaner API
const useUserStore = create((set) => ({
  data: null,
  loading: false,
  fetchUser: async () => {
    set({ loading: true });
    const response = await api.getUser();
    set({ data: response, loading: false });
  }
}));

I propose we start by migrating the Settings module as a testbed. It's isolated enough not to break core flows but complex enough to validate the pattern.

Thoughts on the migration strategy?
